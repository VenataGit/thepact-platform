This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

The user provides frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

### Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction:
- **Purpose**: What problem does this interface solve? Who uses it?
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, etc.
- **Constraints**: Technical requirements (framework, performance, accessibility).
- **Differentiation**: What makes this UNFORGETTABLE? What's the one thing someone will remember?

**CRITICAL**: Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism both work—intentionality matters, not intensity.

Then implement working code that is:
- Production-grade and functional
- Visually striking and memorable
- Cohesive with clear aesthetic direction
- Meticulously refined in every detail

### Frontend Aesthetics Guidelines

#### Typography
- Use distinctive, characterful fonts — avoid overused defaults
- Mix font weights and sizes for hierarchy and visual interest
- Consider letter-spacing, line-height, and text transforms

#### Color & Theme
- Build cohesive palettes with a dominant color and sharp accents
- Use gradients, overlays, and tints for depth
- Avoid clichéd color schemes — find unexpected combinations

#### Motion & Animation
- CSS animations with high-impact moments (page transitions, hover states, loading)
- Micro-interactions that feel responsive and alive
- Scroll-triggered animations for narrative flow

#### Spatial Composition
- Unexpected layouts: asymmetry, overlap, broken grids
- Generous whitespace OR intentional density
- Z-index layering for visual depth

#### Backgrounds & Visual Details
- Atmosphere: textures, patterns, noise, grain
- Depth: shadows, blur, parallax
- Details: borders, dividers, ornamental elements

### Anti-Patterns to Avoid
- Generic AI aesthetics: card grids with rounded corners and blue gradients
- Overused font families (Inter, Poppins without purpose)
- Cookie-cutter layouts with no context-specific character
- Predictable color schemes without intentionality

### Implementation
Interpret creatively with unexpected choices. Match implementation complexity to vision. Think outside the box and commit fully to distinctive vision. Every pixel should feel intentional.
